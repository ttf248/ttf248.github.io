Taking advantage of the Double Eleven promotion, Alibaba Cloud has purchased a new server: an economical version, costing 99 yuan per year. The configuration isn’t high-end, but it’s good as a jump server to proxy services for home use. The promotion lasts until 2026.

I specifically chose a server in the Shanghai area, proxying my home machine with low latency. It’s Windows 11 and Windows Server 2022; the server version was deployed later. When I tried to use it, I suddenly received an “access denied” message. I initially thought it was due to a server update and would return to normal shortly. After waiting five minutes, I tried again, but still couldn’t log in. Searching for related errors indicated that someone was attempting to log in, and now login is unavailable due to too many incorrect password attempts.

I’ve previously dealt with security attack scripts, and immediately thought that these logins are likely not normal behavior – someone is attacking the service, attempting to brute-force the login server. The server firewall settings were convenient but didn’t set up whitelists. It proxied the 3389 ports of two machines, exposing them on the public network—just like bait in a fishpond. Now that I know it’s a script kiddie attacking, the next steps are simple: configure the firewall whitelist to only allow access to the proxy service from company IP addresses and home IP addresses.

The frps agent server wasn’t configured with running logs before. After enabling the logs, I was quite surprised to see proxy IPs from all over the country attempting to log in to my home server. Luckily, one of them was a server version, which alerted me to the problem. Otherwise, that Windows 11 machine would have been compromised eventually – the password was set up pretty simply.
