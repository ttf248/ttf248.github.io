claude code 常用操作指南

2025年11月08日
阅读约 2 分钟
tianlong.xiang
English 日本語

cc 提供的命令那么多，好用的都有哪些，抖音找个了视频学习一波，简单的记录下我认为有用的。

生成规范化 git 日志，默认会带上版权信息，但是国内已经不用他家的模型了，对接的都是国内大模型，git 日志里面的版权也就没必要加上了（github 会展示 cc 为合作开发者）

找到用户配置文件，添加配置项 "includeCoAuthoredBy": false

/init 分析分析当前文件夹生成一份概要，方便大模型更好的理解项目

/compact /clear 日常命令，不解释

claude --dangerously-skip-permissions 慎用，上篇稿子的 null 文件就是这样弄出来的，cc windows 上有很多小问题

# 进入记忆模式，项目级别用的比较少，我常用在用户级别，常用项目要求写到里面

/ide 能感知到 vscode 当前选中的文本数据，trae 里面，经常交互的时候，我是提供很多相关文件的函数片段，切换到命令行，一直没找到类似的功能，主要是很多时候项目的文件很多，制定参考的函数，能有效提供生成代码的准确率

mcp ？一直没怎么尝试此类型的工具，后续有需要再来尝试，老觉得用起来有点麻烦

自定义命令 没需求，用户级别的能替代之前写的 git 规范化递交

-r 关闭项目后重启找到之前的对话，对于复杂的项目，用处不大，国内的模型，上下文 200k，很容易就满了
