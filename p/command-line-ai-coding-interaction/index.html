基于命令行的AI编码交互

2025年11月05日
阅读时长: 7 分钟
tianlong.xiang
English 日本語

惯例打开 Trae 准备开始搞搞代码，通知栏收到消息：claude 模型下线了，不能用，后续大概率也无法恢复，官方给了补偿方案，使用次数增加 300（截止到一月份）。

查了下，不出预料，anthropic 公司按照美国的要求，禁止国内的公司继续使用 claude 系列模型。混入 trae discord 社群，看到不少人吐槽 claude 模型的下线，毕竟大部分人都是冲着 claude 来的。早在 claude 4.5 模型没有在 trae 同步上线，这事就已经有了苗头。

尝试

抱着最后试试的心态，体验了还支持的其他模型，包括 openai 的 gpt-3.5-turbo、gpt-4、google 的 gemini pro。

怎么说呢，效果都不太理想，不知道 trae 境外团队怎么开发的，按理说，不会有那么大的差距。测试用的提示词，还是以前练手项目：小蓝书，以前的文章有写过。

叠加对于 Trae IDE 本身就有不满意，遂发了邮件给 Trae 团队，申请退款。

改变

没记错应该是谷歌先发布的，基于终端交互的 ai 编程，相对 ide，日常的智能提示没有了，但是更通用了，开发人员可以继续使用原来的开发环境。

OpenAI 和 Anthropic 这两家公司，分别发布了 claude code、codex，工具和模型并没有彻底绑定，通过修改配置文件，也能接入其他模型。

discord 社群中，有人提到了 minimax m2 和 glm4，这俩都是国内的模型，还是小蓝书项目，体验了前者，还不错。

安装需要科学上网，不同模型的切换，推荐：https://github.com/farion1231/cc-switch

claude code

依赖 node js，命令：npm install -g @anthropic-ai/claude-code

常用的命令：

  • 使用 /clear 清除聊天上下文，避免累积过多历史信息影响效率
  • claude –dangerously-skip-permissions，执行这个之后，Claude 会自动跳过所有权限确认
