번뜩이는 사이 또 일 년이 지나갔다. 업무적으로 가장 큰 변화는 인공지능의 참여도가 눈에 띄게 높아진 것인데, 이전에는 다양한 개발 언어 간 전환 시 개발자가 익숙해해야 할 여러 언어들의 다른 API 인터페이스들이 있었지만, 이제 이러한 기본적인 코드들은 AI를 통해 생성할 수 있게 되었다. 이는 개발자들에게 큰 행운이다.

ChatGPT

23년도에 간단한 입문 소개 두 편을 다룬 적이 있었는데, 눈 깜짝할 사이에 25년이 벌써 왔네요. 어떻게 말해야 할지 모르겠지만, 뚜렷한 발전은 없는 것 같아요. 아직은 자신만의 인식을 가지고 합리적으로 작업을 분할할 수 있어야 하고, 무엇보다 중요한 것은 AI가 생성한 코드에 버그가 있는지 식별하는 것입니다.

Github copilot

어느 날인지 잊혀졌는데, 자료를 보니 싱가포르에 서버를 구축했고, 이제 국내에서 사용할 수 있어서 더 이상 오랫동안 VPN을 사용할 필요가 없어. 물론 로그인할 때는 아직 VPN이 필요하지만, 이 VPN은 로그인할 때만 잠깐 사용하면 되고 그 후에는 종료해도 돼.

일상적으로 더 많이 사용하는 것은 Github Copilot 플러그인인데, 이 플러그인은 vscode나 visual studio에서 바로 사용할 수 있습니다. 두 프로그램 사이를 전환할 필요가 없습니다. ChatGPT에 비해 Github Copilot이 프로젝트 지원이 더 좋고, 상호작용도 더 친절하며, 일부 로컬 파일을 선택하여 AI를 **“훈련”**시킬 수 있어 생성되는 코드가 프로젝트에 더 잘 맞습니다.

Cursor AI

최근에 새로운 AI 프로그래밍 IDE인 Cursor AI를 보았는데, 이 IDE는 Github Copilot을 기반으로 하고 있지만, 더 똑똑하고 파일을 직접 생성해 줄 수 있습니다

간단히 사용해 봤는데 괜찮은 것 같긴 하지만, 현재 프로젝트에 대한 이해는 아직 부족하고, 로컬 프로젝트 파일이 많을 때는 큰 규모의 리팩토링이나 최적화 조정은 여전히 개발자가 작업을 분할해야 한다

예시 하나를 덧붙입니다: curso의 엔지니어링 모드로 전환하여 다음 내용을 입력하세요: 새로운 개인 이력서 웹페이지를 생성하고, 여러 가지 다른 스타일로 전환할 수 있도록 지원하며, 데이터 표시를 위해 몇몇 개인 정보를 채워 넣으세요

몇 번이고 두었다 떼었다 하는 과정을 거치면 다음과 같은 웹페이지를 얻을 수 있습니다. 물론 이 웹페이지는 비교적 간단하지만, 초보자에게는 아주 괜찮은 시작입니다.

현재 등록 사용자는 고급 API를 150회 무료로 체험해 볼 수 있으며, 유료 사용자는 고급 API 사용이 5000회로 제한됩니다
