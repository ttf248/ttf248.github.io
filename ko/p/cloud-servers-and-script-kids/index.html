더블 11 프로모션으로 알리바이가 새 서버를 구매했습니다. 경제 버전이라 구성은 높지 않지만, 집안 서비스의 게이트웨이 역할을 하기에 좋고, 내년 2026년까지 프로모션이 진행 중입니다.

특别히 상하이 지역 서버를 선택해서 집의 컴퓨터에 낮은 지연 시간 프록시로 연결했는데, win11과 windows server 2022를 사용하고 있습니다. 서버 버전은 나중에 배포되었고, 사용할 때 갑자기 접근 거부 메시지를 받았습니다. 서버 업데이트라고 생각하며 잠시 기다리니 정상적으로 복구될 줄 알았는데, 5분 후에 다시 시도해도 여전히 로그인되지 않습니다. 관련 오류를 검색해 보니 누군가가 로그인을 시도하고 있고, 비밀번호가 틀린 횟수가 너무 많아 현재 로그인이 불가능하다고 합니다.

이전에 보안 공격 스크립트를 다뤄본 경험이 있어 바로 알 수 있었다. 이 로그인 시도는 정상적인 행동이 아니며, 누군가 서비스를 공격하고 로그인 서버를 무차별 대입하려고 하는 것이다. 방화벽 설정은 간단하게 해놓았고, 화이트리스트 설정도 되어 있지 않다. 두 대의 기계의 3389 포트를 프록시했는데, 이 포트가 공공 네트워크에 노출되어 있는 상황이었다. 마치 연못 속 미끼와 같다. 스크립트 키드의 공격이라는 것을 알았으니, 다음 단계는 간단하다. 방화벽 화이트리스트를 설정하여 회사 IP 주소와 집 IP 주소만 프록시 서비스에 접속하도록 허용한다.
