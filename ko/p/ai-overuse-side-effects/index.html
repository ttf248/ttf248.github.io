새로운 __INLINE_CODE_0__이 생긴 이후로, 별 볼 일 없는 것들을 AI로 기록하고 게시하는 데 많은 노력을 쏟고 있는데, 정적이고 스스로 생각하는 시간은 점점 줄어들고 있습니다. 앞으로 이 항목의 생산을 조금 통제하고 월간 형태로 통합하는 것이 좋겠습니다. 매달 한 편씩 발행하면 될 것 같습니다.

이것은 일종의 후유증과 같거나, 부작용이라고 할 수 있는데, 효율은 높아졌지만 사고의 깊이와 넓이는 얕아졌다

효율성 향상: 부정할 수 없다

__INLINE_CODE_0__예전에는 이 코너를 잘 관리하지 않았습니다. 몇몇 주요 이슈에 대해서는 게으름 때문에 인터넷에서 자료를 검색하고 정리하는 일은 하지 않았죠. 요즘은 다양한 AI 도구가 생겨서 개요만 작성하면, AI가 자동으로 관련 정보를 검색하고 기록하여 필요한 글을 생성해 줍니다. 간단히 서식을 정리해서 게시하면 됩니다.

이것은 게으름뱅이를 위한 구세주와도 같아서, 효율이 눈에 띄게 높아졌고, 노력의 절반으로 두 배의 효과를 보는 것 같습니다

스크립트 작성은 잠시 접어두고 코딩할 때 효율이 눈에 띄게 올라갔다. 이전에는 API 인터페이스 문서 설명을 자세히 읽어야 했는데, 이제는 그냥 넘어가도 되는데, 이런 점화는 상당히 유용하다. API 숙지는 __INLINE_CODE_0__BOLD_2脑力劳动의 영역이고, AI가 이 부분을 처리하기에 딱 좋다.

쓰레기 내용

많은 원고들이 있는데, 내용 품질이 높지 않아. 완전히 엉망인 것은 아니지만, 읽으면 활력이 없고, 예전부터 싫어했던 스타일이야. 밋밋하고 재미없어.

AI가 생성한 콘텐츠는, 바꿔 말하면, 마치 조립 라인에서 나온 것처럼 영혼이 부족하다고 할 수 있습니다

망각성

이런 종류의 초고는 AI가 생성한 것이고, 독자 상황은 파악하기 어렵지만, 시간이 지나면 제 기억조차 흐릿해지거나 잊혀질 수도 있습니다

비슷한 문제가 코딩할 때도 발생하는데, 코드 커밋 기록을 확인하지 않으면 당시 어떻게 생각했는지, 왜 그렇게 썼는지 전혀 알 수 없다. 특히 코드 검토와 AI와의 반복적인 소통 끝에 생성된 코드는 처음의 생각과 크게 달라서 심지어 완전히 다른 경우도 있다.

검색

최근에 구글이나 바이두를 열어보는 횟수가 눈에 띄게 줄었다. AI로 검색하면 상호작용도 좋고 검색 결과도 전통적인 검색 엔진보다 훨씬 낫다.

지금은 생존 여부가 불분명한 __INLINE_CODE_0__에게 경의를 표하며, 대기업 내에서 가장 먼저 출시된 인터넷 검색 기능이 탑재된 AI 도구입니다

구글을 잘 안 쓰게 됐고, INLINE_CODE_0 접속 횟수도 줄어들었어. 많은 문제를 AI에 바로 물어볼 수 있어서 이 사이트는 점점 시대에 뒤쳐질 것 같아.
