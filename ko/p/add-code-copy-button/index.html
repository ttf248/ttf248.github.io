서론

원저작자의 초고를 꼼꼼히 살펴보고, 한 번 쭉 읽은 다음 작가 소개도 살펴보자. 와, 대단한 분을 만났네. 칭화대 재학 중인 학부생인데, 어릴 때부터 컴퓨터에 익숙했었나 보네. 아, 갑자기 뭘 해야 할지 기억이 안 나. 관련해서는 저자의 Github 저장소도 한번 살펴보고, 이 수정된 even 테마가 지금보다 훨씬 보기 좋고 기능도 많아. 이걸 해봐야겠다. 일단 관련된 코드를 가져와서 합쳐보자.

새로운 기능: 기사 이력 확인, 연관 제출 기록 보기

효과는 괜찮은 편이고, 기사 맨 끝부분에서 체험해 볼 수 있습니다

손질하기 전에 작가의 원 저장소 기록을 자세히 보지 않았고, 간단하게 병합하면 될 줄 알았는데, 결국 많은 코드를 병합했고, 중간에 충돌도 여러 번 있었고, 무식하게 덮어썼다. 내가 원하는 대로 하기 위해서 프론트엔드와 렌더링 템플릿 코드들이었다.

저장소 주소: https://github.com/TianlongXiang/hugo-theme-even

중국어의 함정인데, git에서 이 파라미터를 조정하지 않으면 현재 문서의 commit hash를 가져올 수 없어 역사 링크 생성이 실패합니다. 전체 역사를 생성할 때도 자동 통합 스크립트를 수정해야 하고, 현재 저장소의 모든 역사 기록을 가져오도록 해야 합니다.

  • 기능: 전체 GitHub 저장소 가져오기를 시도하여 기사 마지막 수정 기록을 동적으로 업데이트합니다
  • 경로에 중국어가 존재하므로, Hugo GitInfo 설정에서 이 기능을 활성화해야 합니다
name: Build Github
run: git config --global core.quotePath false && hugo -b "https://www.xiangtianlong.com/" -d "github_public" && ls

본문

ouuan의 코드 기록을 한 시간 반 넘게 봤는데, 복사 버튼을 어떻게 추가하는지 도통 이해가 안 간다

시간이 흘러 한 달쯤 지나서, 또 이 일이 생각났다

이 과제가 이해가 안 되면 다른 문제로 베끼고, 어쨌든 따라 하면 알 수 있을 거야. 검색 결과는 꽤 뜻밖이었는데, hugo 공식 포럼에 복사 버튼을 추가하는 방법에 대한 글이 있길래 가봤더니 논리가 명확하더라고. 그런데 이상한 일이 벌어졌어. 돌아와서 보니 even으로 렌더링된 코드 블록 스타일이 자료 설명과 달라서 이 부분은 좀 번거로워. 간단히 기록해 둘게.

기본적인 프론트엔드 개발 지식이 부족해서 모르는 부분은 브라우저의 요소 검사를 열어 코드와 오른쪽 스타일 정보에 맞춰 분석하며 천천히 로직을 이해하게 되었고, JavaScript는 이해가 안 가서 콘솔에 로그를 남겼다. 많이 모를 때는 차분히 마음을 가다듬고 천천히 로직을 정리하고 분해하면 결국 해결책이 보인다.

pre 노드가 여러 개 있는데, 여기서은 단일 코드 블록을 의미하며, 테마에서 자체적으로 라인 번호를 렌더링하여 복사 버튼이 두 개로 나타납니다 이런, 관심 주제 자체의 코드 하이라이트 렌더링을 구현하고 싶지만 이 주제 설정에 익숙하지 않네요 공식 홈페이지에서 자료를 봤는데, 반쯤 이해가 됐고, 코드 하이라이트를 제어할 수 있는 마크업 설정이 있다는 걸 알게 됨 설정 파일을 조정해도 계속 안 되고, 렌더링 결과가 예상과 다릅니다 이 설정들 pygmentsOptions을 발견하고 또 자료를 뒤져서 설정을 조정했다. 우선 행 번호는 제거한다. 사용자 정의 css 스타일 시트와 사용자 정의 js 스크립트를 구성합니다 이렇게까지 많은 일을 해놓았으니, 문득 최근에 봤던 색상 조합이 괜찮다는 생각이 들어서 버튼 스타일을 조금 바꿔볼까 하는데: 중국풍 천청색으로 해보자
