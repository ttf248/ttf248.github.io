이주

퇴근하면서 짐을 정리하고, 익숙하게 자리를 옮겨 새로운 자리에서 컴퓨터 배선을 조정하니 편안한 자세로 일하기 딱 좋네요

(ÒωÓױ)！ 인터넷 연결했는데, 팀에서 주로 사용하는 서버에 접속이 안 돼요. 무선 네트워크로 바꿔보니 다시 정상적으로 접속되네요.

처음에는 서버 네트워크 세그먼트 설정 문제라고 생각했습니다. 새로운 작업 공간의 유선 네트워크는 방화벽 구성 목록에 없어서 IT 동료에게 연락해서 조정하면 될 거라고 생각했죠. 그런데 이 네트워크 세그먼트는 여러 대의 서버가 있는데, 다른 서버들에 접속해 보니 모두 정상이었고, 점차 의심이 들기 시작했습니다. 전문적인 일은 전문가에게 맡기는 것이 좋다고 생각하고, 결국 운영 부서 동료들이 문제 해결을 위해 조사한 결과, 해당 서버에 docker가 배포되어 있어서 서비스의 기본 네트워크 docker0와 사무실 유선 네트워크 구성 세그먼트 간 충돌이 발생하여 전송된 데이터 패킷이 응답을 받지 못하고 docker 서비스로 라우팅되었다는 것을 확인했습니다.

다른 서버에는 docker 서비스가 배포되어 있지 않아서, 이 서버만 사용하고 있고 가끔 컨테이너로 테스트 서비스를 배포하곤 했는데, 이런 상황을 마주할 줄은 몰랐다. 나중에 생각해보니, 그룹 전체가 같은 사무실 건물에 있고 IT 부서 동료들이 네트워크 세그먼트를 분할하면서 172로 시작하는 주소를 사용한 것은 이상한 일이 아니라고 생각했다.

docker0 

# vim /etc/docker/daemon.json
{
    "bip":"172.200.0.1/24"
}

서비스를 재시작하고 새로운 네트워크로 전환하면 서버가 정상적으로 접근 가능해집니다
