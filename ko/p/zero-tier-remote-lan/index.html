지리적 분산 환경의 제로티어

사무실에 새 mini 호스트를 구입했는데, 환경 설정이 간편하고 가끔 집에서 접속할 필요가 있어서 임시로 내장 네트워크 침투를 구축했습니다. 이전 경험을 바탕으로 frp 서비스를 배포하고 포트 전달을 지정했더니 공용 서버가 필요했고 연결 품질은 공용 서버의 대역폭에 따라 달라졌습니다. 새로운 Zerotier 가상 머신 로컬 영역 네트워크를 구축했는데, 이는 VPN과 유사하며 로컬에서 가상 네트워크 카드를 새로 만들고 모든 장치를 하나의 가상 네트워크에 추가했습니다.

사무실에 새 mini 호스트를 구입했는데, 환경 설정이 간편하고 가끔 집에서 접속할 필요가 있어서 임시로 내장 네트워크 침투를 구축했습니다. 이전 경험을 바탕으로 frp 서비스를 배포하고 포트 전달을 지정했더니 공용 서버가 필요했고 연결 품질은 공용 서버의 대역폭에 따라 달라졌습니다. 새로운 Zerotier 가상 머신 로컬 영역 네트워크를 구축했는데, 이는 VPN과 유사하며 로컬에서 가상 네트워크 카드를 새로 만들고 모든 장치를 하나의 가상 네트워크에 추가했습니다.

제로티어는 무엇인가요?

ZeroTier는 다양한 지리적 위치의 장치 간에 안전한 가상 네트워크를 생성할 수 있는 소프트웨어 정의 광역 네트워크(SD-WAN) 솔루션입니다. ZeroTier를 통해 여러 대의 컴퓨터, 서버 및 장치를 마치 동일한 로컬 영역 네트워크에 있는 것처럼 하나의 가상 암호화된 네트워크에 쉽게 연결할 수 있습니다. 이를 통해 프로그래머와 IT 전문가는 복잡한 네트워크 설정 및 VPN 구성 없이 서로 다른 위치에서 데이터와 리소스를 안전하게 공유할 수 있습니다.

제로티어 네트워크는 가상 전역 로컬 영역 네트워크로, 서로 다른 장치들이 인터넷을 통해 연결되어 마치 동일한 물리적 네트워크에 있는 것처럼 작동하도록 합니다. 이 네트워크는 여러 서브넷으로 구성될 수 있으며, 모든 장치는 제로티어 기술을 통해 연결됩니다.

행성 서버: 행성 서버는 Zerotier 네트워크의 핵심 구성 요소 중 하나입니다. 이들은 전 세계적으로 운영되며, Zerotier 네트워크 전체의 토폴로지 구조, 라우팅 정보 및 네트워크 상태를 유지 관리합니다. 행성 서버는 네트워크 제어 센터 역할을 하지만 데이터를 직접 전송하지는 않습니다. 사용자의 장치는 Zerotier 네트워크에 참여하기 위해 최소한 하나의 행성 서버에 연결해야 합니다.

중계 서버: 중계 서버는 Zerotier 네트워크에서 보조 노드로, 장치 간의 직접 통신 채널을 설정하는 데 도움을 줍니다. 장치가 서로 직접 연결할 수 없을 때, 이들은 중계 서버를 통해 데이터를 전송할 수 있습니다. 이는 네트워크의 가용성과 성능을 향상시키는 데 도움이 됩니다. 중계 서버는 일반적으로 전 세계 곳곳에 위치하여 데이터 전송의 중계 지점 역할을 합니다.

전반적으로, Zerotier는 행성 서버와 중계 서버의 도움을 받아 전 세계적으로 가상 로컬 영역 네트워크를 구축하여 장치 간 안전하고 빠른 통신을 가능하게 합니다. 행성 서버는 전체 네트워크 관리를 담당하며, 중계 서버는 필요할 때 장치가 통신 연결을 설정하도록 돕습니다.

설치 배포

ZeroTier 공식 웹사이트(https://www.zerotier.com/)에서 설치 파일과 문서를 다운로드하세요 귀하의 운영 체제에 따라 ZeroTier One 클라이언트 프로그램을 다운로드하여 설치하십시오. Windows, macOS, Linux 및 기타 여러 플랫폼을 지원합니다. 설치가 완료되면 ZeroTier One 클라이언트를 시작합니다 ZeroTier 계정이 아직 없으시다면, 계정을 만드세요. 클라이언트에서 계정을 생성할 수 있습니다. ZeroTier 계정으로 로그인하여 새 네트워크를 만드세요. 네트워크에는 고유한 16자리 ID가 있으며, 이 ID를 기억해야 합니다. 귀하의 장치에 이 네트워크를 추가합니다. 클라이언트에서 네트워크 ID를 입력하거나 QR 코드 스캔 기능을 사용할 수 있습니다. 7. ZeroTier 클라이언트가 설치 및 구성된 장치는 동일한 가상 네트워크에 추가됩니다. 이제 이러한 장치들은 마치 동일한 로컬 영역 네트워크에 있는 것처럼 직접 통신할 수 있습니다. ZeroTier 제어판에서 네트워크 설정 관리, 장치 추가 및 네트워크 트래픽 모니터링 등을 할 수 있습니다

문 설치 배포

국내 통신사들은 대부분 UDP 포트 포워딩을 금지하고 있습니다. frp 서비스는 안정적이며, TCP 프로토콜을 사용하기 때문에 Zerotier를 통해 리퍼 서버를 배포하여 유사한 효과를 낼 수 있습니다. 방화벽에서는 udp 9993 포트를 개방해야 합니다.

curl -s https://install.zerotier.com/ | sudo bash

설치 성공 여부 확인

zerotier-cli info

로컬 네트워크에 참여하세요

zerotier-cli join network-id

moon을 생성합니다

cd /var/lib/zerotier-one && sudo zerotier-idtool initmoon identity.public > moon.json

편집기 구성 파일을 조정하고 stableEndpoints 노드를 “서버 공용 IP/9993"으로 설정합니다

서명 구성 생성, moons.d 폴더 생성 후 해당 파일들을 이동하고 서비스를 재시작합니다

sudo zerotier-idtool genmoon moon.json
mkdir moons.d && mv 000000eb444ec0d8.moon moons.d/
systemctl restart zerotier-one.service

클라이언트 노드가 문(moon) 서버에 접속하며, ID는 앞선 JSON 설정 파일의 ID 필드에서 가져옵니다

zerotier-cli.bat orbit ztaddr ztaddr

Windows 플랫폼에서는 관리자 권한으로 터미널을 실행하고 zerotier-cli.bat 명령줄 도구를 사용하여 Linux 플랫폼에서는 zerotier-cli를 사용합니다. peers 서브 명령어는 연결 상태를 확인하고, listpeers는 모든 노드를 표시하며, moon 노드가 정상적으로 표시되면 성공적으로 조인된 것입니다.

어떻게 제거할 수 있나요

윈도우 플랫폼에서 제거하는 방법은 더 이상 설명하지 않겠습니다. 일반적인 작업으로 제어판을 통해 제거하면 됩니다. 우리는 Ubuntu에 대해 중점적으로 이야기하겠습니다.

dpkg를 통해 zerotier-one 서비스를 삭제합니다

sudo dpkg -P zerotier-one

zerotier-one 폴더를 삭제하면 주소가 저장되어 있는데, 삭제 후 다시 설치하면 새로운 주소를 얻을 수 있습니다

sudo rm -rf /var/lib/zerotier-one/

사후기

원래 다 이미 제거해 놨는데, 서버는 도착했는데 적절한 프록시 노드 역할을 할 서비스가 없어서 알리바바 클라우드가 실적을 위해 개발 전용 서버를 제공했어. 사양은 높지 않고, 연간 99위안으로 저렴해서 두 대 정도 돌리고 있어. 서버에서 제공하는 대역폭이 마음에 들어서 그런 거야.
